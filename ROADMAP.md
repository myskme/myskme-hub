# MYSKME · 作品演进路线图

> 核心判断：作品库重心倒挂——6 个作品 5 个在“激励/参与”轴，全部“学习重量”压在 1 个（题库训练场），而它只把**词汇**练扎实；中考卷面却是**阅读+听力+写作**主导。
> 结论：下一步不是第 7 个 app，而是**把已造好的引擎指向还没覆盖的技能**，并先加固承重缝（题库契约 / 版本控制 / 进度可携）。

排序依据：价值 × 工时（详见会话中的优先级矩阵）。

## 近期 · 1–2 周（低成本高回报：夯地基 + 点燃留存）
- [x] **① 冻结题库 JSON schema + 校验器**（`/schema`）——出卷端与网站端共用，出库前 `python3 schema/validate_tiku.py 卷号.json`。✅ 2026-06-24
- [ ] **学习核心入 git**：把《词灵对决》+《无名之原》（现拖拽部署、无源码）抢救进版本控制。
- [ ] **② 词灵对决：连胜火苗 + 今日任务 + 7 日格子**（引擎已有，补动机层）。
- [ ] **③ 卷尾 QR 自带兑换码** `?code=XXXX`，一扫直达当晚复习、题库预加载。
- [x] **④ 语法陷阱 SRS deck**：八类陷阱喂给词灵对决同一引擎（同引擎换内容）。✅ 2026-06-24 → `banks/TRAP1.json`（32 题，过 3 评审答案唯一性审计）
- [ ] **⑤ 家长周报卡**：一键导出“本周练 X 词 / 攻克 Y 错题 / 连续 Z 天”（复用海报 canvas）。
- [ ] **⑥ 全站“MYSKME 世界”回链页脚**：六座孤岛变漏斗。

## 中期 · 1–2 月
- [ ] **⑦ 进度可携**：名字+短码作 key，搬家改为一个 QR（解决跨设备连胜重置）。
- [ ] **⑧ 试卷→题库“双产物默认”**：出卷同一次运行吐 PDF + 合规题库 JSON+兑换码。
- [ ] **⑨ 词灵对决换 MYSKME 世界皮**（词灵=黑域生物、avatar=琳）。
- [ ] **⑩ 收敛 3 积分器 → 1**（积分板降为内部“纯数据”模式，第三套冻结；换肤+模式切换软着陆，不删）。
- [ ] **反向错词回流**：词灵对决导出“本周错词”短码，出下份卷子粘回（同时减备课负担）。
- [ ] **段落级阅读题型**：无名之原渲染 200–300 词文章 + 题组 + 逐题解析。

## 长期 · 季度+
- [ ] **⑪ 听力训练场（server-free）**：音频作 `/listening` sibling 文件，听力题走同一份题库 JSON；先 TTS 预渲染降人力。补最大单一计分缺口。
- [ ] **⑫ 中考作文训练场（单文件）**：选题→三段式脚手架→草稿→范文+自评 rubric；作文作为题库一种题型。
- [ ] **收集 / 赛季层**（卡牌词灵 + 赛季相册）；**hub 升级“一屏方法故事”**（吸引其他老师采用）。
- [ ] **抽共享 myskme-core**（theme/tokens/水印/QR helper）build 时内联，停止 7 文件手工复制品牌。

## 资源整合
- 一份 JSON 契约统管：兑换码 / 学生身份短码 / 题库（词灵+无名共用）。
- 一个 SRS 引擎服务多技能：词汇 → 语法陷阱 →（未来）听力/阅读。
- 一套 canvas 导出复用：海报 → 家长周报卡 → 赛季相册。
- 一条管线双向跑：试卷→题库（去）+ 错词短码→出卷（回）。
- 一套 IP 喂多处：叶王/黑域/琳 → 词灵皮肤 + 大乱斗 + 无名之原。

- ❌ 把远征录/编年史当“学习或参与”继续投入——记为品牌资产、冻结新开发，只收割 lore 喂词灵对决。
- ❌ 把试卷最富的语法/阅读信号压平进通用题池。
- ❌ 四处手工镜像作品清单（让 build_hub.py 当唯一源）。
