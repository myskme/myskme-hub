## 2. 信息架构 · 五个视图

顶部：`授业历` 题字（serif 金字）+ 学期档案徽章（如「暑历 · 7.6–8.31」）+ 视图切换。
桌面 = 顶部切换；移动 = 底部 Tab。快捷键：`T` 回今天，`W/D/M/S` 切视图，`←/→` 翻周，`N` 快速排课。

### 2.1 今日（移动端默认落地页）
- **Hero 卡**：正在上的课（大字学生名 + 剩余分钟进度条）或下一节课倒计时（「距 小明·初二语法 还有 1 小时 20 分」）。无课日显示留白雅句（如「今日无课 · 磨刀亦是授业」）。
- 下方：今天时间轴（竖向），已完成的课盖金印 ✓，未打卡的过去课显示「待落印」琥珀提醒（TutorBird overdue attendance 思想）。
- 底部速览：本周已上 N 节 / 待上 M 节。

### 2.2 周视图（桌面默认）
- 7 列（周一…周日，周一起始），纵轴默认 08:00–22:00（设置可改），30 分钟一格、15 分钟吸附。
- 今日列：淡金水洗底 + 列头日期加金印徽记。now-line：金色渐变横线 + 左端菱形标记，每分钟重算。
- 课块：左侧 4px 学生色带 + serif 学生名 + 小字主题 + 右上状态章（✓金印=完成 / 假=请假 / 移=改期，图标+文字双编码，不只靠颜色——可访问性）。
- 同一时段重叠（家教罕见）：并排分栏，>3 个收纳成「+N」。
- 顶栏：`◀ 本周 ▶` + 周区间文字 +「回到今天」。
- 交互见 §4。

### 2.3 月视图（热力总览）
- 月历格：每格金色深浅 = 当天课时数（Timepage Month Heatmap），格内小数字 = 节数；点格 → 弹当日课程列表 → 可跳周视图。
- 用途：一眼看暑假排布密度、某生本月频率。

### 2.4 学生档案
- 学生卡列表：色标 + 姓名 + 年级 + **课时余额**（「余 6 次」；≤2 时琥珀高亮「该续了」——Teachworks 余额预警）+ 待补课标记（「欠补 1 节」）。
- 点开 = 单生页：基本信息（年级/上课方式/费率(可选)/备注）、固定课规则列表、**历史课时间线**（倒序：日期 + 内容 + 作业 + 表现星 + 关联题库码）——王老师课前 30 秒回顾上次讲了什么（调研：个人老师最痛的点）。
- 课时包：充值记录数组（+N 次/小时，备注），余额 = 充值 − 消耗（消耗按 status 自动算）。计量单位每生可选「按次/按小时」。

### 2.5 统计
- 本周/本月：完成节数、总课时（小时）、按学生分布（横向条，学生色）。
- 课酬：`费率 × 课时` 汇总，**默认隐藏**，点 👁 才显示（投屏/旁人在场不漏钱数）。
- 记录连击：连续 N 天完成打卡（复用全站 streak 语感）。
- 月末一句话总结卡（如「七月：授业 42 节 · 63 小时 · 最勤 小明 12 节」），可导出 PNG 备忘（复用海报 canvas 套路，P2）。

## 3. 数据模型（契约，localStorage 键 `myskme-almanac-v1`）

```json
{
  "v": 1,
  "settings": {
    "dayStart": "08:00", "dayEnd": "22:00", "snapMin": 15,
    "showMoney": false, "lastExport": "2026-07-04"
  },
  "terms": [
    { "id": "t1", "name": "暑历", "from": "2026-07-06", "to": "2026-08-31" },
    { "id": "t2", "name": "秋学期", "from": "2026-09-01", "to": "2027-01-15" }
  ],
  "students": [
    { "id": "s1", "name": "小明", "grade": "初二", "color": "#c9a64a",
      "mode": "线下", "unit": "次", "rate": 0, "note": "",
      "packages": [ { "date": "2026-07-01", "amount": 10, "note": "暑假包" } ] }
  ],
  "rules": [
    { "id": "r1", "studentId": "s1", "weekday": 2, "start": "16:00", "end": "17:30",
      "topic": "初二语法", "from": "2026-07-06", "to": "2026-08-31", "termId": "t1" }
  ],
  "overrides": [
    { "id": "o1", "ruleId": "r1", "date": "2026-07-14", "action": "cancel", "note": "学生请假" },
    { "id": "o2", "ruleId": "r1", "date": "2026-07-21", "action": "move",
      "newDate": "2026-07-22", "newStart": "10:00", "newEnd": "11:30" },
    { "id": "o3", "ruleId": null, "date": "2026-07-18", "action": "once",
      "studentId": "s1", "start": "14:00", "end": "15:30", "topic": "补课·从句", "makeup": true }
  ],
  "logs": [
    { "id": "l1", "key": "r1@2026-07-07", "date": "2026-07-07", "studentId": "s1",
      "start": "16:00", "end": "17:30",
      "status": "done",
      "content": "定语从句 which/that 辨析", "homework": "CLAUSE1 配对 18 题",
      "bankCode": "CLAUSE1", "stars": 4, "privateNote": "" }
  ]
}
```

要点（执行时严格遵守）：
- **课的身份 key**：规则实例 = `ruleId@date`；一次性课 = override 的 `id`。log 用 `key` 关联，排课与留痕解耦（改排课不丢记录）。
- `weekday`：1=周一 … 7=周日。日期一律 `YYYY-MM-DD` 本地日期字符串（**不要用 Date 直接序列化**，避免时区漂移）。
- `status` 枚举与课时结算：`done`（扣）/ `leave_makeup` 学生请假·补（不扣，学生补课额度 +1）/ `leave_burn` 学生请假·耗（扣）/ `teacher_cancel`（不扣）/ 改期不是 status，是 override。`makeup:true` 的一次性课完成时核销一个补课额度。
- **某周渲染算法**：① 展开该周内所有生效 rules（`from<=date<=to`，termId 若填则再校验日期落在该 term）命中的日期 → 实例；② 按 `ruleId+date` 应用 overrides：cancel 删、move 改（newDate 在本周则移入/移出）；③ 叠加本周内 `action:"once"` 的一次性课；④ 按 key join logs 得到状态章。
- 迁移：读档时 `if (data.v < CURRENT) migrate(data)`，预留升级钩子。
- **备份环**：每次写档若距上个快照跨天，把整档推入 `myskme-almanac-bak`（数组，保留最近 10 份，带日期）。设置页可一键回滚某快照。距 `lastExport` 超过 14 天时顶部出现一条低调金边提示「两周没导出备份了」。

## 4. 交互规格

### 4.1 排课（最高频，三条通道全做）
1. **点空白格原地内联建课**（Outlook Inline Creation）：点格 → 格内直接出现输入框，输入学生名（已有学生自动联想，新名字即建新学生并自动分色）→ 回车成课（默认 1.5h，可拖底边改）。连排多节最顺手。
2. **拖拽建课**：空白处按下拖出时长 → 松手出迷你确认卡（学生 + 主题）→ 确认。产生半透明预览块，吸附 15 分钟。
3. **快速一句话排课**（`N` 键 / 移动端 ➕，Fantastical 自然语言的确定性简化版，纯正则不猜）：
   - `小明 周二四 16:00-17:30 初二语法` → 建 2 条循环规则（默认当前学期档案区间）
   - `明天 15:00 小明 作文` → 一次性课（默认时长 1.5h）
   - `7.18 14:00-15:30 小明 补课` → 一次性课，含「补课」自动 `makeup:true`
   - 解析要素：星期集合（周X 可连写）/ 今天·明天·后天 / `M.D` 日期 / `HH:MM(-HH:MM)` / 学生名（对照已有学生表匹配）/ 剩余文字=主题。解析结果先回显确认再落库。

### 4.2 调课
- 拖块到新位置（跨天/改时段皆可）→ 松手弹**两档选择**：「仅这一次」（生成 move override）/「此后每周」（改 rule，并把 rule.from 断代：旧 rule 截止昨天、新 rule 今起——保历史正确）。一次性课直接改。
- 移动端：长按块进入拖动；或点块 → 详情卡里「改期」。

### 4.3 消课（灵魂动作，两次点击内完成）
- 点任意「已过时/进行中」的课块 → 详情卡置顶一排大按钮：`✓ 落印完成` `假·补` `假·耗` `师·取消`。
- 点「落印完成」→ **金印微动效**（印章缩放盖下 + 一圈墨晕淡开，0.5s，庄重不糖果——Morgen 完成庆祝的黑金版）→ 自动生成 log 并**带入该生上次记录的骨架**（上次主题/作业结构预填，My Music Staff 笔记延续性），王老师增量改两笔即存。
- log 表单四件：本次内容 / 作业（可填题库兑换码，自动生成 `word-duel?code=` 深链）/ 表现 1–5 星 / 私密备注。全部可空——**先落印后补记也行**，不阻塞。
- 今日视图对「过去未打卡」的课持续显示「待落印」琥珀章，点即打卡（防忘）。

### 4.4 冲突与护栏
- 建课/调课落库前查重叠：同时段已有课 → 红边提示「与 小明 16:00 冲突」，可强行保存（家教可能故意连堂）。
- 删除学生：若有历史 log，只允许「归档」（隐藏于列表，记录保留）；无记录才真删。
- 所有破坏性操作（删规则/删快照/清空）二次确认。

## 5. 视觉规格（黑金编年史 · 与全站同源）

- **执行前先读 `myskme-game-art` 技能**（反单调军规 + SVG 纹理配方）。token 直接抄 `console/index.html`（深色黑金 + 浅色羊皮纸两套 `--bg/--ink/--gold/--line/--page-grad/--vignette` 等已全备），主题控件 = 全站统一 `◐ 跟随系统→浅→深`，写 localStorage **`myskme-theme`**（同源全站同步，别新造键）。
- 字体：标题/学生名 serif（`--serif` 栈），时间刻度/数字 `--mono`。
- 网格质感：纵向时间刻度线用 `--line2` 细线，整页羊皮纸/墨底纹理 + vignette；周末两列在学期档案下加极淡金染（暑假档案下取消，七天平权——呼应「暑假全周」）。
- 课块：`--card-bg` 渐变 + 学生色带 + hover 抬升（translateY(-2px) + `--card-hover-shadow`）；完成课右上金印章 SVG（篆刻风「印」字或狼首徽记）。
- 空状态必须设计（军规：不留白板）：无学生 → 卷轴插画 +「立第一位门生」；本周无课 → 雅句。
- 移动端 ≥44px 触控目标；两主题全部文案过 WCAG AA（沿用 console 校过的 ink3/ink4/gold 数值）。
- **品牌铁律**：页面文案、注释、导出物一律无 AI 字样；页脚署名 `MYSKME · 狼先生与他的学生们`。

## 6. 学期档案（双模式）规格

- terms 数组可增删改（设置页）；**按今天日期自动选中**当前档案，顶栏徽章显示（「暑历」/「秋学期」）；点徽章可临时预览其他档案区间（学期没结束就先排暑假课——调研建议的「自动优先 + 手动兜底」）。
- rule 可绑 termId（该档案区间内才展开），也可不绑（长期有效，跨档案）。
- 暑假结束切学期：不用重排——暑假 rules 自动过期，学期 rules 自动生效。
- 种子数据：首次打开引导建两个档案（暑历 2026-07-06→08-31 / 秋学期 09-01→2027-01-15，可改）。

## 7. 打印与导出

- **A4 横向周课表**（`@media print`，沿用 hub 打印模式套路）：黑白线框、去底色阴影、隐藏 now-line 与一切交互件；顶部印「授业历 · YYYY.MM.DD–MM.DD · MYSKME」，字号 ≥10pt，右上留手写批注空白。浅色羊皮纸基底 token 强制（`print-color-adjust`）。
- JSON 导出/导入（全档）：文件名 `almanac-YYYY-MM-DD.json`；导入前校验 `v` 字段并自动备份现档。
- ICS 导出（P2）：把未来 8 周展开实例导出 VEVENT，可进系统日历/手机提醒。

## 8. 明确不做（个人版护栏，别加）

多老师/多教室/权限、家长端门户或推送、群聊、审批流、取消计费阈值规则引擎、在线支付/发票、招生 CRM、班课容量——全部机构级，个人纯本地工具零收益高复杂度（调研三路一致结论）。家长沟通 = 截图/打印周表即可。

## 9. 跨设备（两台电脑 + 手机的现实）

- P0/P1：数据在各设备 localStorage 独立；靠「JSON 导出 → 存 iCloud `新英语/` → 另一台导入」+ 14 天导出提醒。诚实、零后端。
- P2 可选「**云档**」：仿名人天梯经验开一个**独立**极小 Cloudflare Worker（一个职责：按口令哈希 GET/PUT 单个 JSON blob ≤256KB，不长成教师后台——遵守 roadmap 对 Worker 的克制令）。做不做由用户到时拍板；学生名继续用化名即无 PII 顾虑。

## 10. 技术与文件结构

- `almanac/index.html` 单文件（vanilla JS，零依赖，零 CDN），预上色主题脚本放 `<head>`（防闪白，抄 console 前 10 行）。
- 状态管理：单一 `DB` 对象 + `save()` 节流写 localStorage；渲染纯函数按视图重绘；`setInterval` 每分钟刷 now-line/今日 Hero。
- 课块绝对定位：`top = (start − dayStart) / 总分钟 × 100%`，`height = 时长 / 总分钟 × 100%`。
- 大文件审查经验：单文件会到 100KB+，review/agent 阅读时**分段读**（gotcha 已知）。

## 11. 分期计划与验收

**P0（一次会话可完成，上线即可用）**
外壳+软门+noindex+双主题 → 数据层（模型/展开算法/备份环/导出导入）→ 周视图（网格/课块/now-line/今日列）→ 今日视图 → 学生管理（含课时包余额）→ 排课通道 1（内联建课）+ 调课两档 + 消课打卡（含金印动效、log 表单、课时结算）→ 冲突检测 → 空状态。
✅ 验收：新档案从零建 2 生 3 规则 → 翻 4 周正确展开；请假/改期/补课各走一遍余额正确；刷新/重开数据在；两主题 AA；iPhone 390px 可完整操作；口令门拦截未授权。

**P1（第二会话）**
月热力图、统计面板（含 👁 课酬开关）、快速一句话排课、拖拽建课/调课、补课额度核销、上次记录骨架预填、「待落印」提醒。

**P2（按需）**
A4 打印、ICS 导出、月度总结 PNG 卡、云档同步（用户拍板后）、console/hub-admin 入口接线（改 `build_hub.py`，**推送需用户明示授权**）。

**上线前 QA（每期必做）**：跑一轮多审校对抗式审查（正确性/结算逻辑/时区日期/两主题对比度/移动端五个方向），确认后再交付；浏览器实测消课全状态矩阵（5 status × 按次/按小时）。
