# MYSKME 题库格式契约 · v1

MYSKME 题库工坊统一题库（“扩展题库”）格式。**出卷端**（试卷即题库管线）产出、**网站端**（《词灵对决·进阶版》/《无名之原》）导入，双方共用这一份契约。本目录就是这份契约的唯一权威来源。

> 稳定地址（可在出卷端与网站端引用）：
> `https://myskme.github.io/myskme-hub/schema/tiku.schema.json`

## 文件
| 文件 | 作用 |
|---|---|
| `tiku.schema.json` | JSON Schema（draft-07），给编辑器 / CI 用 |
| `validate_tiku.py` | **零依赖校验器**，出库前命令行跑一遍 |
| `sample-S1E0.json` | 合规示例，可直接照抄结构 |

## 格式速览
```jsonc
{
  "ver": 1,
  "name": "S1E7 派对藏宝图",     // 题库名
  "code": "S1E7",               // 兑换码 = 卷号（字母数字下划线，2–16 位，学生手输）
  "desc": "一句话简介",
  "date": "2026-06-24",
  "mods": {                      // 模块表，键 m1/m2/m3…
    "m1": [ ["abandon", "v. 抛弃；放弃", 2] ],   // 配对题：[词, 义, 难度?]
    "m4": [ {                                    // 选择/填空题
      "q": "She ___ her dream.",   // 空格写 ___
      "ops": ["abandoned", "achieved", "avoided", "admired"],
      "ans": 1,                    // 正确项下标：0 起，且必须 < ops 长度
      "why": "achieve one's dream = 实现梦想。",
      "lv": 2                      // 1=基础 2=冲刺（可省）
    } ]
  }
}
```

**判定规则（与线上导入器逐条一致）**
- 每题按**形状**区分：`数组` = 配对题，`对象` = 选择/填空题（模块键 m1/m2/m3 习惯放配对，m4/m5 放选择，但不强制）。
- `ans` 是 **0 起下标**，必须 `0 ≤ ans < ops.length`。
- `why`（解析）必填、不能为空。**答案串必须与教师解析版逐位核对一致后才算完成。**
- `code`（兑换码）= 卷号，如 `S1E7`，永不另起名。

## 出库前校验（必做）
```bash
python3 schema/validate_tiku.py 你的卷号.json
# 绿勾 ✓ 才发布；红叉 ✗ 会逐条指出 mod 键 + 第几题 + 错在哪
```

## 预留字段（未来题型，当前导入器忽略，不影响现有题库）
选择题对象上预留：`type`（题型，缺省=选择/填空）、`media`（听力音频指针，如 `{"src":"audio/S1E7-1.opus","dur":12}`）、`passage`（段落级阅读语段）、`rubric`（作文自评量规）。新增**听力 / 阅读 / 作文**模块时沿用本 schema，只填这些字段，无需 v2 迁移。
